# Backend Folder Structure

This document explains the organization of the F2X_NeuroHub backend directory.

## Root Directory Files

Essential configuration and entry point files only:

- `main.py` - Application entry point
- `pyproject.toml` - Python project configuration
- `pytest.ini` - Pytest configuration
- `alembic.ini` - Alembic database migration configuration
- `README.md` - Main project documentation
- `.env.example` / `.env.template` - Environment variable templates
- `.gitignore` - Git ignore patterns
- `uv.lock` - UV package manager lock file

## Main Directories

### `/app`
Main application code containing:
- `/api` - API endpoints (v1)
- `/core` - Core functionality (config, security, dependencies)
- `/models` - SQLAlchemy ORM models
- `/schemas` - Pydantic schemas for validation
- `/services` - Business logic layer
- `/crud` - Database operations (if separated from services)

### `/alembic`
Database migration files managed by Alembic

### `/database`
Database-related utilities and seeders

### `/tests`
Unit and integration tests

### `/scripts`
Utility and maintenance scripts:
- Database management scripts (`init_db.py`, `seed_db.py`, `manage_db.py`)
- Migration scripts (`migrate_*.py`)
- Diagnostic scripts (`check_*.py`, `diagnose_*.py`)
- Fix/patch scripts (`fix_*.py`, `add_*.py`)
- Verification scripts (`verify_*.py`, `test_*.py`)
- SQL scripts (`*.sql`)

### `/docs`
Project documentation:
- `N+1_OPTIMIZATION_SUMMARY.md` - Query optimization documentation
- `OPTIMIZATION_SUMMARY.md` - General optimization documentation
- `REFACTORING_SUMMARY.md` - Refactoring history
- `SETUP_ALEMBIC.md` - Alembic setup guide
- `FOLDER_STRUCTURE.md` - This file

### `/logs`
Application and debug logs:
- `backend_debug.log`
- `backend_logs.txt`
- `tracker_*.log`
- Other runtime logs

### `/monitoring`
Monitoring and observability tools

### `/htmlcov`
Code coverage HTML reports (generated by pytest-cov)

### `/.pytest_cache`
Pytest cache directory

### `/.venv`
Python virtual environment (not committed to git)

## Database Files (Development)

SQLite database files for local development:
- `dev.db` - Main database file
- `dev.db-shm` - Shared memory file
- `dev.db-wal` - Write-ahead log file

> **Note**: These files are gitignored and should not be committed.

## Best Practices

1. **Keep root clean**: Only essential configuration files in the root directory
2. **Use subdirectories**: Organize related files into appropriate subdirectories
3. **Document changes**: Update this file when adding new major directories
4. **Follow conventions**: Place files in their designated directories based on their purpose
